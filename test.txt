Here's the exact list of changes I made to turn your source into a working static-export build. Apply these in your office repo, then run the build and zip the out/ contents.
Files I changed (3 files)
1. next.config.ts — switch from server build to static export, and disable image optimization (the optimizer needs a server, which static hosting doesn't have).
Replace the whole file with:
tsimport type { NextConfig } from "next";

const nextConfig: NextConfig = {
  output: "export",
  images: { unoptimized: true },
};

export default nextConfig;
2. src/app/page.tsx — the root page used redirect(), which needs a server. Static export has none, so make the redirect run in the browser instead.
Replace the whole file with:
tsx"use client";
import { useEffect } from "react";
import { useRouter } from "next/navigation";

export default function Home() {
  const router = useRouter();
  useEffect(() => {
    router.replace("/dashboard/overview");
  }, [router]);
  return null;
}
3. src/app/layout.tsx — this one is a special case. I only changed it because my build machine couldn't reach Google Fonts. On your office machine, you do NOT need this change — keep your original layout.tsx with next/font/google, because the font downloads fine there during the build.
Only use the version below if your office build also fails with a "Failed to fetch Manrope from Google Fonts" error (e.g. behind a strict office firewall/proxy). It loads Manrope via a normal stylesheet link instead:
tsximport "../styles/globals.css";
import AosProvider from "@/src/components/animation/AosProvider";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <head>
        <link rel="preconnect" href="https://fonts.googleapis.com" />
        <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
        <link
          href="https://fonts.googleapis.com/css2?family=Manrope:wght@300;400;500;600;700&display=swap"
          rel="stylesheet"
        />
      </head>
      <body style={{ fontFamily: "Manrope, system-ui, sans-serif" }}>
        <AosProvider>{children}</AosProvider>
      </body>
    </html>
  );
}

What changed: output goes from "standalone" to "export" (produces a static out/ folder), and images are set to unoptimized because static hosting has no server to optimize them. The images.unsplash.com remote pattern is no longer needed once optimization is off.
File 2 — chryselys-pandora-frontend/src/app/page.tsx
BEFORE (original — uses a server-side redirect):
tsximport { redirect } from "next/navigation";

export default function Home() {
  redirect("/dashboard/overview");
}
AFTER (browser-side redirect, because static export has no server):
tsx"use client";
import { useEffect } from "react";
import { useRouter } from "next/navigation";

export default function Home() {
  const router = useRouter();
  useEffect(() => {
    router.replace("/dashboard/overview");
  }, [router]);
  return null;
}
What changed: redirect() from next/navigation runs only on a server. In export mode there's no server, so it's replaced with a client component that redirects in the browser using useRouter().replace inside useEffect.
File 3 — chryselys-pandora-frontend/src/app/layout.tsx
This one you only change if your office build can't reach Google Fonts. On a normal machine, keep the BEFORE version.
BEFORE (original — loads Manrope at build time via next/font/google):

  children,

      </body>

AFTER (loads Manrope via a stylesheet link in the browser — only needed if the build host is blocked from Google Fonts):
tsximport "../styles/globals.css";
import AosProvider from "@/src/components/animation/AosProvider";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <head>
        <link rel="preconnect" href="https://fonts.googleapis.com" />
        <link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
        <link
          href="https://fonts.googleapis.com/css2?family=Manrope:wght@300;400;500;600;700&display=swap"
          rel="stylesheet"
        />
      </head>
      <body style={{ fontFamily: "Manrope, system-ui, sans-serif" }}>
        <AosProvider>{children}</AosProvider>
      </body>
    </html>
  );
}
What changed: the build-time font fetch (next/font/google) is removed and Manrope is loaded by the visitor's browser via a <link> tag, so the build no longer needs Google Fonts access.
